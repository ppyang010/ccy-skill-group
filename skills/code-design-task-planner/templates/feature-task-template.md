## 维护规则

- 每次开始一个明确任务前，确认或补充对应任务项。
- 每次完成一个明确任务后，把对应条目标记为 `[x]` 并补完成时间。
- 新增任务时，记录添加时间，例如：`（添加时间：2026-05-20）`。
- 任务完成时，记录完成时间，例如：`（添加时间：2026-05-20，完成时间：2026-05-20）`。
- 已有未带时间的历史任务保持不变，不强制补时间。
- 每次发现新的阻塞项，补到“当前阻塞项”。
- 每次确定新的开发重点，更新“建议的下一步”。
- 只有真正完成的事项才移动到“已完成事项”。
- 出现需求目标、边界、接口、字段、数据结构、实现顺序、验收口径、任务优先级或范围变动时，先同步更新 `{{功能名}}-desgin.md`，再同步本文档。
